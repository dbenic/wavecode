# Server Setup Guide

WaveCode runs on any Linux server (or macOS machine) with SSH access. This guide covers provisioning a dedicated server from scratch — applicable to any provider (Hetzner, OVH, Vultr, DigitalOcean, AWS, your own hardware, etc.).

## Minimum Requirements

| Resource | Minimum | Recommended | Notes |
|----------|---------|-------------|-------|
| CPU | 4 cores | 8+ cores | Agents are mostly I/O-bound (waiting on LLM APIs), so cores matter less than you'd think |
| RAM | 16 GB | 64 GB | Each agent uses ~2-3 GB (Node.js + tmux + git) |
| Storage | 100 GB SSD | 500 GB+ NVMe | Git worktrees add up fast — NVMe strongly recommended |
| Network | 100 Mbit/s | 1 Gbit/s | Agents call LLM APIs constantly |
| OS | Ubuntu 22.04+ / Debian 12+ | Ubuntu 24.04 LTS | Any modern Linux with systemd works |

**Rough capacity planning:**

| RAM | Agents | Monthly cost (typical VPS) |
|-----|--------|---------------------------|
| 16 GB | 5-8 | $10-20 |
| 32 GB | 8-15 | $20-50 |
| 64 GB | 15-25 | $40-100 |
| 128 GB | 25-50 | $100-250 |

## Step 1: Provision a Server

Order a server from any provider. Dedicated servers offer the best value for always-on agent fleets; cloud VMs work well for staging or smaller setups.

When choosing:
- **Dedicated servers** — best price/performance for running agents 24/7
- **Cloud VMs with dedicated vCPUs** — good middle ground, easy to resize
- **Shared vCPU instances** — fine for testing or small fleets (agents are mostly idle)

Install **Ubuntu 24.04 LTS** (or Debian 12+) as the OS.

## Step 2: Secure the Server

```bash
# Connect as root
ssh root@YOUR_SERVER_IP

# Create a non-root user
adduser wavecode
usermod -aG sudo wavecode

# Set up SSH key authentication
mkdir -p /home/wavecode/.ssh
# Paste your public key (from your local machine: cat ~/.ssh/id_ed25519.pub)
echo "YOUR_SSH_PUBLIC_KEY_HERE" >> /home/wavecode/.ssh/authorized_keys
chown -R wavecode:wavecode /home/wavecode/.ssh
chmod 700 /home/wavecode/.ssh
chmod 600 /home/wavecode/.ssh/authorized_keys

# Disable root login and password auth
sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
systemctl restart sshd

# Firewall — allow only SSH, block everything else
apt update && apt install -y ufw
ufw default deny incoming
ufw default allow outgoing
ufw allow ssh
ufw enable
```

> If you plan to use Tailscale, also run `ufw allow in on tailscale0` after installing Tailscale.

## Step 3: Install Prerequisites

```bash
# Switch to your user
su - wavecode

# System packages
sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential git tmux curl

# Node.js 22 LTS
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# Verify
node --version   # v22.x
npm --version    # 10.x
tmux -V          # tmux 3.x
git --version    # git 2.x
```

## Step 4: Install WaveCode

```bash
# Clone (replace with your repo URL)
git clone https://github.com/YOUR_ORG/wavecode.git ~/wavecode
cd ~/wavecode

# Run the installer
./scripts/install.sh
```

The installer will:
- Install Node.js dependencies
- Build the UI
- Create `config.yaml` from the example
- Generate an access token
- Optionally install a systemd service
- Add shell aliases

## Step 5: Configure

Edit `~/wavecode/config.yaml`:

```yaml
server:
  port: 3777
  host: 0.0.0.0

paths:
  projects_root: ~/projects     # where your code lives

auth:
  method: token                 # or 'tailscale' if using Tailscale
  fallback_token: YOUR_TOKEN    # generated by the installer

# Add your LLM API keys for research, command chat, etc.
llm:
  provider: anthropic
  anthropic_api_key: sk-ant-...
  openai_api_key: sk-...
```

See [config.example.yaml](../config.example.yaml) for all options including local LLM setup.

## Step 6: Start WaveCode

```bash
# Foreground (for testing)
wavecode server start --foreground

# Background (via tmux)
wavecode server start

# Or via systemd (recommended for production)
sudo systemctl start wavecode
sudo systemctl status wavecode
```

Open `http://YOUR_SERVER_IP:3777` in your browser and enter your access token.

## Step 7: Network Access (Optional)

WaveCode handles app-level auth. Securing the network path is up to you.

### Option A: Tailscale (Recommended)

The simplest way to access WaveCode from your phone or laptop without exposing it to the internet:

```bash
# Install Tailscale
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

# Allow Tailscale through the firewall
sudo ufw allow in on tailscale0
```

Then access WaveCode at `http://YOUR_TAILSCALE_HOSTNAME:3777` from any device on your tailnet.

### Option B: SSH Tunnel

```bash
# From your local machine
ssh -L 3777:localhost:3777 wavecode@YOUR_SERVER_IP

# Then open http://localhost:3777 locally
```

### Option C: Reverse Proxy (Nginx + HTTPS)

```bash
sudo apt install -y nginx certbot python3-certbot-nginx

# Create nginx config
sudo tee /etc/nginx/sites-available/wavecode << 'EOF'
server {
    server_name wavecode.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:3777;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;          # important for SSE
        proxy_cache off;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/wavecode /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# Get HTTPS certificate
sudo certbot --nginx -d wavecode.yourdomain.com

# Allow HTTP/HTTPS through firewall
sudo ufw allow 'Nginx Full'
```

## Step 8: Start Using Agents

```bash
# SSH into your server
ssh wavecode@YOUR_SERVER_IP

# Start a coding agent in tmux
tmux new -s cl-backend
claude   # or: codex --full-auto / aider --yes

# Detach (Ctrl+B, D), then adopt in WaveCode
wavecode scan
wavecode adopt cl-backend

# Or use the web dashboard: Scan → Adopt
```

## Updating WaveCode

```bash
cd ~/wavecode
git pull
./scripts/install.sh     # re-runs build, preserves config.yaml
sudo systemctl restart wavecode  # if using systemd
```

## Troubleshooting

### Server won't start
```bash
# Check logs
journalctl -u wavecode -f          # systemd logs
cat ~/wavecode/wavecode.db          # database exists?
node --version                      # Node 22+?
```

### Agents not detected
```bash
tmux list-sessions                  # are sessions running?
wavecode scan                       # does scan find them?
```

### Permission errors
```bash
# Ensure wavecode user owns everything
chown -R wavecode:wavecode ~/wavecode
chmod 600 ~/wavecode/config.yaml    # config has API keys
```

### Out of memory
```bash
# Check per-agent memory usage
ps aux --sort=-rss | head -20

# Add swap as emergency buffer
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

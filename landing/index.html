Three steps to parallel agents

From zero to a fleet of coding agents in minutes.

1

Adopt or Spawn

Discover existing tmux sessions or create new ones. Zero disruption to running agents.

$ wavecode scan

Scans for existing tmux sessions running Claude Code, Aider, Codex, or any CLI. Adopt them in-place without restarting.

$ wavecode spawn --runtime aider --model claude-sonnet

Or spawn fresh agents with the runtime and model of your choice. WaveCode handles the tmux session lifecycle.

Click to expand +
2

Define Goals or Tasks

Describe what you want. AI decomposes into a DAG pipeline with dependencies.

$ wavecode goal "Add auth to the API and write tests"

WaveCode uses AI to decompose your goal into sub-tasks with a dependency graph. Backend first, then tests, then integration -- all automatic.

Or define granular tasks manually and wire dependencies yourself. Full control when you want it.

Click to expand +
3

WaveCode Orchestrates

Auto-dispatch to idle agents, monitor progress, handle retries, manage artifacts.

Tasks are dispatched to idle agents respecting the dependency DAG. WaveCode monitors progress via capture-pane or structured events.

Failed tasks get retried. Completed work is reviewed (auto or manual). Artifacts are shared between agents. Notifications hit your phone.
